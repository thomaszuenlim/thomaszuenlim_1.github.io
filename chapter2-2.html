Changelog #2: Initial Budget

Throughout user testing in Paper Prototype 1, we quickly realised that for several features in the Pay and Spending screen, the user would need to be able to set up a budget during their onboarding process.

The addition of an initial budget setup screen after the Tutorial screen allowed us to include essential budgeting information alongside Pay, Spend and Track.




After testing our High Fidelity version 1, we realised that the addition of accounts in the onboarding process had become fundamentally overcomplicated.

The previous implementation of account setup required users to select all of the financial accounts they'd like to add upfront, only to then be allowed set them all up consecutively.
i.e. If the user want to add a Personal Banking account, a Superannuation account and an Afterpay account; they would have to:

1. Select Personal banking
2. Select Superannuation
3. Select Afterpay
4. Press next to proceed to setting up each of these accounts; one after another.

This raised several concerns, like what if one of those accounts had an issue? How would we consider the error handling in a task that fails halfway through?

It was at this point that we realised that we needed to fundamentally change the order of this process. Instead of selecting all your accounts and then setting them up, to increase instancy and responsiveness, we decided to allow users to select a specific account, and then immediately start to set it up. Once successfully connected, the user can return back to the account selection page and continue to add the other accounts that are of interest to them.

Changelog #4: Confirmation Prompts

The final wireflow change can be found in the confirmation behaviour introduced to major account changes.

For example, when users go to change their budget, they are now required to confirm that these changes are to be made, to ensure that users are always aware that they are about to make a financial alteration to their accounts.
